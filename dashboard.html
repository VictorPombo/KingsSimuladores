Arquitetura — Site Unificado
1 site, 1 deploy, 1 domínio

UI / Design System
Componente "StreamingBackground"
Design padrão das páginas: fundo animado ultra-leve (sem filtros CPU bound) otimizado para 60fps cravados.
💰
Pagamentos
Mercado Pago
Checkout transparente (Pix/Boleto/Cartão 12x) + Split Payment para marketplace.
🧾
NF-e — Centralizada
100% via Olist
As emissões de notas fiscais de venda serão engatilhadas de forma passiva direto para o ERP Olist para centralizar tudo lá nativamente.
📦
Omnichannel & ERP (Olist/Bling)
Integração via Hub Homologado
A KingsHub centraliza o cadastro e aciona a API do Olist. O Olist assume a burocracia e distribui os anúncios em massa (Shopee, ML, Amazon). O fluxo de volta usa um webhook do Olist para dar baixa no estoque da KingsHub.
🛡️
Validação de Produtos
Regra Rígida de Cadastro
O formulário de produto na KingsHub exige preenchimento obrigatório e travado de NCM, EAN, Peso e Dimensões. O sistema impede o salvamento se faltar algum dado, evitando erros de rejeição nas APIs dos marketplaces.

Quick Start — Ordem Exata de Execução Seguir em ordem

⚠️ Importante: Execute os passos abaixo nesta ordem exata. Cada passo depende do anterior. Não pular etapas.

Passo 1 — Banco
Rodar SQL no Supabase
Abrir SQL Editor no Supabase e rodar os 5 blocos SQL acima em ordem. Verificar que as 11 tabelas aparecem no Table Editor.
Passo 2 — Monorepo
Inicializar Turborepo + 3 apps
Criar estrutura monorepo com apps/kings-store, apps/msu-marketplace, apps/admin e os packages compartilhados (ui, db, config, utils, payments, shipping, notifications).
Passo 3 — .env.local
Preencher variáveis de ambiente
Criar .env.local na raiz com todas as variáveis da seção acima. Valores do Supabase: Dashboard → Settings → API. MP: painel do Mercado Pago.
Passo 4 — Supabase Client
Configurar packages/db
Instalar @supabase/supabase-js e @supabase/ssr. Criar client para server components, client components e middleware separadamente.
Passo 5 — Auth
Configurar Supabase Auth
No Supabase Dashboard → Authentication → Providers: habilitar Email, Google e Apple. Configurar URLs de redirect (localhost + domínios de produção).
Passo 6 — Design System
Criar @kings/ui
Package compartilhado com Tailwind v4, tokens de cor/spacing/tipografia, componentes base. Disponível nos 3 apps via workspace.
Passo 7 — Vercel
Configurar os 3 projetos
Importar repo no Vercel 3 vezes. Em cada projeto: Root Directory = apps/kings-store (ou msu-marketplace, ou admin). Adicionar todas as env vars.
Passo 8 — Verificar
Teste de ponta a ponta
Rodar turbo dev local. Registrar usuário → confirmar que profile foi criado no Supabase pelo trigger → fazer logout → login → checar sessão.
